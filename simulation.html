ℹ️ How to play — goal & rules

The goal. Turn a storm of random code fragments into a working program. Every ⚡ strike runs whatever the clouds have collected — you're aiming for as many green output lines (real, useful output) as possible, and as few red errors. There's no score and nothing to lose: it's a sandbox — watch order appear out of chaos, and nudge it along.

Nudge it. ⚡ Strike = run what's collected now · 🌬 Gust, ＋ Cloud, or click the sky = stir up more collisions · click a cloud = change its code.

Make it grow. 🧠 Memory lets variables survive between strikes, so the program builds up · 🎯 Coverage guarantees the basic variables exist, so fewer lines error out.

The twist — the whole point. Turn on 🧬 Evolution and watch for a minute. The system starts optimizing “survival”: clouds get great at not-erroring while useful output collapses toward zero. The metric gets gamed. That live effect is exactly what the Reward Hacking Lab is built on. Hit 🧪 Run 100 for the exact, repeatable numbers (same seed every time).
